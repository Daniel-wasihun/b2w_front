# Plan: Schema and Admin Dashboard Entity Design

## Overview
Comprehensive design document for the Born ToWin (B2W) platform admin dashboard and database schema.

---

## 1. Database Schema (Laravel Backend)

### 1.1 Core Entities

#### Users
| Field | Type | Description |
|-------|------|-------------|
| id | bigint (PK) | Primary key |
| name | string | User full name |
| email | string (unique) | Email address |
| email_verified_at | timestamp | Verification timestamp |
| password | string | Hashed password |
| department_id | bigint (FK) | Department reference |
| is_active | boolean | Account status |
| remember_token | string | Auth token |
| created_at | timestamp | Creation time |
| updated_at | timestamp | Last update |

Relationships:
- `belongsTo(Department)`
- `belongsToMany(Role)` via role_user pivot
- `hasMany(Submission)`
- `hasMany(Participant)`
- `hasMany(Certificate)`

#### Roles
| Field | Type | Description |
|-------|------|-------------|
| id | bigint (PK) | Primary key |
| name | string | Role name (Admin, User, etc.) |
| slug | string | URL-safe identifier |
| description | text | Role description |
| permissions | json | Permission flags |
| created_at | timestamp | Creation time |
| updated_at | timestamp | Last update |

Pivot Tables:
- `role_user`: role_id, user_id

#### Departments
| Field | Type | Description |
|-------|------|-------------|
| id | bigint (PK) | Primary key |
| name | string | Department name |
| slug | string | URL-safe identifier |
| description | text | Department details |
| created_at | timestamp | Creation time |
| updated_at | timestamp | Last update |

Relationships:
- `hasMany(User)`

#### Categories (Race Categories)
| Field | Type | Description |
|-------|------|-------------|
| id | bigint (PK) | Primary key |
| name | string | Category name |
| slug | string | URL-safe identifier |
| description | text | Category details |
| icon | string | Icon class |
| created_at | timestamp | Creation time |
| updated_at | timestamp | Last update |

Relationships:
- `hasMany(Race)`

#### Races (Competitions)
| Field | Type | Description |
|-------|------|-------------|
| id | bigint (PK) | Primary key |
| category_id | bigint (FK) | Category reference |
| title | string | Race title |
| slug | string | URL-safe identifier |
| description | text | Race details |
| image_url | string | Cover image |
| start_date | timestamp | Start date |
| end_date | timestamp | End date |
| max_participants | integer | Capacity limit |
| status | enum | active, upcoming, completed, archived |
| created_at | timestamp | Creation time |
| updated_at | timestamp | Last update |

Relationships:
- `belongsTo(Category)`
- `hasMany(Participant)`
- `hasMany(Submission)`
- `hasMany(Certificate)`

#### Participants
| Field | Type | Description |
|-------|------|-------------|
| id | bigint (PK) | Primary key |
| race_id | bigint (FK) | Race reference |
| user_id | bigint (FK) | User reference |
| status | enum | active, completed, withdrawn |
| progress | integer | Completion % (0-100) |
| joined_at | timestamp | Join timestamp |
| completed_at | timestamp | Completion time |

Relationships:
- `belongsTo(Race)`
- `belongsTo(User)`

#### Submissions
| Field | Type | Description |
|-------|------|-------------|
| id | bigint (PK) | Primary key |
| race_id | bigint (FK) | Race reference |
| user_id | bigint (FK) | User reference |
| title | string | Submission title |
| description | text | Submission details |
| file_url | string | Uploaded file |
| status | enum | pending, approved, rejected |
| score | decimal(5,2) | Score (0-100) |
| feedback | text | Reviewer feedback |
| submitted_at | timestamp | Submission time |
| reviewed_at | timestamp | Review time |
| created_at | timestamp | Creation time |
| updated_at | timestamp | Last update |

Relationships:
- `belongsTo(Race)`
- `belongsTo(User)`

#### Certificates
| Field | Type | Description |
|-------|------|-------------|
| id | bigint (PK) | Primary key |
| race_id | bigint (FK) | Race reference |
| user_id | bigint (FK) | User reference |
| submission_id | bigint (FK) | Submission reference |
| verification_code | string | Unique verification hash |
| issued_at | timestamp | Issue date |
| certificate_url | string | PDF download link |
| status | enum | active, revoked |
| created_at | timestamp | Creation time |
| updated_at | timestamp | Last update |

Relationships:
- `belongsTo(Race)`
- `belongsTo(User)`
- `belongsTo(Submission)`

#### Settings
| Field | Type | Description |
|-------|------|-------------|
| id | bigint (PK) | Primary key |
| key | string | Setting key |
| value | text | Setting value |
| description | text | Setting details |
| type | enum | string, integer, boolean, json |

## 2. Admin Dashboard Entity Design

### 2.1 Entity: Users
**List View Columns:**
- Name (with avatar)
- Email
- Department
- Roles (badges)
- Status (active/inactive badge)
- Last login
- Actions (edit, delete)

**Form Fields:**
- Name (text, required)
- Email (email, required, unique)
- Department (select)
- Roles (multi-select)
- Password (password, optional on edit)
- Is Active (toggle)

**Filters:**
- Department
- Role
- Status
- Search (name, email)

**Actions:**
- Create
- Edit
- Delete (soft delete)
- Export CSV
- Bulk actions (activate/deactivate)

### 2.2 Entity: Departments
**List View Columns:**
- Name
- Description (truncated)
- User count
- Created at
- Actions

**Form Fields:**
- Name (text, required)
- Slug (auto-generated)
- Description (textarea)

### 2.3 Entity: Categories
**List View Columns:**
- Name
- Icon
- Description (truncated)
- Race count
- Actions

**Form Fields:**
- Name (text, required)
- Slug (auto-generated)
- Description (textarea)
- Icon (select)

### 2.4 Entity: Races (Competitions)
**List View Columns:**
- Title
- Category
- Image thumbnail
- Status (badge)
- Participants count

- Actions

**Form Fields:**
- Title (text, required)
- Slug (auto-generated)
- Category (select, required)
- Description (rich text)
- Image (upload)
- Start date (datetime)
- End date (datetime)

**Filters:**
- Category
- Status

- Search

### 2.5 Entity: Participants
**List View Columns:**
- User (name, email)
- Race
- Status (badge)
- Progress (progress bar)

- Actions

**Filters:**
- Race

- Status

### 2.6 Entity: Submissions
**List View Columns:**
- Project name
- Race
- User
- Submitted date
- Score

- Actions

**Form Fields (Review):**

**Filters:**
- Race
- User

- Date range

### 2.7 Entity: Certificates
**List View Columns:**
- User
- Race
- Issue date

- User

## 3. API Endpoints Structure

### 3.1 Base Path: `/api/v1/admin`

#### Users
- `GET /users` - List (paginated, filterable)
- `POST /users` - Create
- `GET /users/{id}` - Read
- `PUT/PATCH /users/{id}` - Update
- `DELETE /users/{id}` - Delete (soft)
- `POST /users/bulk-action` - Bulk operations

#### Departments
- `GET /departments` - List
- `POST /departments` - Create
- `GET /departments/{id}` - Read
- `PUT/PATCH /departments/{id}` - Update
- `DELETE /departments/{id}` - Delete

#### Categories
- `GET /categories` - List
- `POST /categories` - Create
- `GET /categories/{id}` - Read
- `PUT/PATCH /categories/{id}` - Update
- `DELETE /categories/{id}` - Delete

#### Races
- `GET /races` - List (paginated, filterable)
- `POST /races` - Create
- `GET /races/{id}` - Read
- `PUT/PATCH /races/{id}` - Update
- `DELETE /races/{id}` - Delete

#### Participants
- `GET /participants` - List (paginated, filterable)
- `POST /participants` - Create (enroll)
- `DELETE /participants/{id}` - Remove

#### Submissions
- `GET /submissions` - List (paginated, filterable)
- `GET /submissions/{id}` - Read
- `PUT/PATCH /submissions/{id}/review` - Review/Score

#### Certificates
- `GET /certificates` - List (paginated, filterable)
- `POST /certificates/{id}/generate` - Generate
- `POST /certificates/{id}/revoke` - Revoke

#### Roles
- `GET /roles` - List
- `POST /roles` - Create
- `GET /roles/{id}` - Read
- `PUT/PATCH /roles/{id}` - Update
- `DELETE /roles/{id}` - Delete

#### Stats/Dashboard
- `GET /stats/overview` - Dashboard overview statistics
- `GET /stats/participants` - Participant trends
- `GET /stats/submissions` - Submission trends

## 6. Type Definitions (TypeScript)

### 6.1 Entity Interfaces

```typescript
// User
interface User {
  id: number;
  name: string;
  email: string;
  email_verified_at: string | null;
  department_id: number;
  is_active: boolean;
  department?: Department;
  roles: Role[];
  created_at: string;
  updated_at: string;
}

// Role
interface Role {
  id: number;
  name: string;
  slug: string;
  description: string;
  permissions: string[];
  users_count?: number;
  created_at: string;
  updated_at: string;
}

// Department
interface Department {
  id: number;
  name: string;
  slug: string;
  description: string;
  users_count?: number;
  created_at: string;
  updated_at: string;
}

// Category
interface Category {
  id: number;
  name: string;
  slug: string;
  description: string;
  icon: string;
  races_count?: number;
  created_at: string;
  updated_at: string;
}

// Race
interface Race {
  id: number;
  category_id: number;
  title: string;
  slug: string;
  description: string;
  image_url: string;
  start_date: string;
  end_date: string;
  max_participants: number;
  status: 'active' | 'upcoming' | 'completed' | 'archived';
  category?: Category;
  participants_count?: number;
  submissions_count?: number;
  created_at: string;
  updated_at: string;
}

// Participant
interface Participant {
  id: number;
  race_id: number;
  user_id: number;
  status: 'active' | 'completed' | 'withdrawn';
  progress: number;
  race?: Race;
  user?: User;
  joined_at: string;
  completed_at: string | null;
}

// Submission
interface Submission {
  id: number;
  race_id: number;
  user_id: number;
  title: string;
  description: string;
  file_url: string;
  status: 'pending' | 'approved' | 'rejected';
  score: number | null;
  feedback: string | null;
  race?: Race;
  user?: User;
  submitted_at: string;
  reviewed_at: string | null;
  created_at: string;
  updated_at: string;
}

// Certificate
interface Certificate {
  id: number;
  race_id: number;
  user_id: number;
  submission_id: number;
  verification_code: string;
  race?: Race;
  user?: User;
  submission?: Submission;
  issued_at: string;
  certificate_url: string;
  status: 'active' | 'revoked';
  created_at: string;
  updated_at: string;
}
```

## 9. Performance Considerations

- Eager loading for relationships (N+1 prevention)
- Database indexing on foreign keys and search fields
- Pagination (25-50 items per page)
- Caching for frequently accessed data (roles, departments)
- Lazy loading for images
- Debounced search inputs
- Optimized queries with `select()` specific fields
